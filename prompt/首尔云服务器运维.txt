## 二、 Web 网页端与 API 服务 (Nginx 统一接管)

本模块包含 **Alpha 加密页面 (`crypto`)**、**带单检测页面 (`copy-sweep`)** 以及它们**共享的 Python 后端 API**。

### 1. 整体架构 (多子域名分发)

* **链路走向:** 用户访问不同子域名 ➔ Cloudflare (仅 CDN/HTTPS) ➔ Nginx (服务器80端口) ➔ 对应前端目录或本地后端。
* **Nginx 路由规则:**
* **前端 `crypto`:** 匹配根目录，返回 `/var/www/alpha-web/`
* **前端 `copy-sweep`:** 匹配根目录，返回 `/var/www/copy-sweep-web/`
* **共享后端:** 带有 `/api/` 的请求，统一转发至本地 `8000` 端口处理。



### 2. 核心文件与配置路径

| 模块 | 文件/目录路径 | 维护场景说明 |
| --- | --- | --- |
| **Alpha 前端代码** | `/var/www/alpha-web/` | `crypto` 域名的静态文件存放处 |
| **Sweep 前端代码** | `/var/www/copy-sweep-web/` | `copy-sweep` 域名的静态文件存放处 |
| **Python 共享后端** | `/root/alpha-api/alpha_api.py` | 修改 AI 提示词、业务逻辑、接口处理 |
| **统一 Nginx 配置** | `/etc/nginx/sites-available/alpha-momentum` | 统一管理上述域名的前端路由及接口转发 |
| **后端系统服务** | `/etc/systemd/system/alpha-api.service` | 修改后端运行端口或环境变量 |

### 3. 标准运维与通用部署流程

🟢 **通用前端网页更新 (Vite 打包部署标准 4 步)**
所有前端页面的更新，请将下方命令中的 `<对应目录>` 替换为实际路径（如 `alpha-web` 或 `copy-sweep-web`）：

1. 本地打包：`npm run build`
2. 清空旧代码：`rm -rf /var/www/<对应目录>/*`
3. 上传新代码：将本地 `dist` 文件夹内的**所有内容**上传至 `/var/www/<对应目录>/`
4. 修复文件权限：
```bash
chown -R www-data:www-data /var/www/<对应目录>
chmod -R 755 /var/www/<对应目录>

```



🟢 **后端逻辑更新 (代码修改后必做)**
修改了 `alpha_api.py` 后，必须重启服务并检查日志：

```bash
systemctl restart alpha-api.service
journalctl -u alpha-api.service -n 50 -f  # 按 Ctrl+C 退出

```

🔵 **Nginx 配置重载 (改路由/域名后必做)**

```bash
nginx -t
systemctl reload nginx

```

#### 2. 常用运维命令（免拼接，直接复制）

**🟢 Lite 策略 (主干) 专用命令**

* **查看状态:** `systemctl status binance-trade-grid`

---

#### 3. 核心注意事项

* **重载配置:** 每次修改配置后，必须先执行以下重载命令，再执行对应的重启命令：
`systemctl daemon-reload`
* **内存保护:** 两套策略均已设定 `MemoryMax=800M`，合计占用不超过 1600M，防止 2GB 服务器内存溢出死机。

## 五、 常见故障排查 (避坑指南)

### 🚨 故障 1：前端更新后浏览器依旧白屏 / 无法显示新功能

* **根本原因：** Cloudflare 边缘节点或浏览器存在“死缓存”。
* **解决方案：**
1. 登录 Cloudflare 控制台 -> Caching (缓存) -> Configuration -> 点击 **`Purge Everything`**。
2. 浏览器端按 `Ctrl + F5` 强制刷新，或在 F12 Network 中勾选 `Disable cache`。
