2. **最差值 (Worst / Min / Max)**：这套参数在最倒霉的时间点入场时的极限抗压能力。

* 样本量暴增：假如周围有 27 个邻居参数组合，乘以 5 个 Offset，你的样本池就变成了 **135 个回测结果**。
* 你在这个包含 135 个不同参数微调、不同入场时间的数据池中，计算 `Mean` 和 `Std`。
* **联合 LCB = (时空大均值) - 1.5 * (时空大标准差)**。


* **逻辑**：这是对抗过拟合的究极形态。如果一个参数得分很高，说明**不仅你稍微改一点参数它还能赚，哪怕你同时把入场时间延后几个小时，它依然稳如泰山！**

#### 预处理层：数据降维与多维特征提取 (Data Fusion)

2. **最差极限 (Worst)**：如 5 个 Offset 中最大回撤的最小值（最深回撤），胜率的最低值。
3. **偏移颠簸度 (Offset_CV)**：核心指标（如收益率、Sortino）在 5 个 Offset 之间的变异系数 (标准差/均值)。

#### L1_HEALTH：时间偏移硬底线 (Worst-Case Survival)

#### L4_NEIGHBORHOOD：3D 参数平原测试 (Spatiotemporal Plateau)

*原逻辑：计算参数附近邻居的存活率和 LCB。*
**新逻辑：时空双重平原下置信界（3D LCB）。**
你现在可以计算一个极其强大的 LCB 分数：

1. **空间（参数）**：原有的逻辑，参数的变动（如窗口从 36 变 40）会带来波动。
2. **时间（Offset）**：同参数下，Offset 的变动也会带来波动。

*这样算出来的 LCB，不仅不怕参数微调失效，也不怕实盘时 K 线换线时间的微妙偏差。*

1. **分离“市场Alpha”与“数据噪音”**

3. **极大增强置信度**
你原本的回测样本路径只有 1 条。引入 5 个 Offset 后，你在不改变历史数据总长的情况下，通过“相位平移”创造了 5 条高度相关但不完全相同的平行宇宙资金曲线。能在这 5 条曲线的交集中存活的参数，其统计学置信度（Confidence Interval）将呈指数级上升。

# 多时间偏移融合的截面动量策略终极稳健性评价方案
## 一、核心设计哲学与底层逻辑
### 1.1 时间偏移测试的本质意义
您引入的`['0h', '30min', '1h', '2h', '3h']`5个时间偏移是**加密货币量化策略反过拟合的黄金标准**，其核心价值在于：
- 破解"K线时间锚定过拟合"：4小时K线的起始时间是人为划分的，真正的市场规律不应依赖于精确的0点/4点/8点等整点
- 模拟实盘执行延迟：实盘中信号触发、订单提交、成交确认必然存在时间差，偏移测试直接模拟了这种不确定性
- 过滤噪声依赖：如果一个参数仅在0h偏移下表现优异，其他偏移下大幅回撤，说明其盈利来源于历史数据中的特定时间点噪声，而非普遍存在的动量效应

### 1.2 多偏移融合的核心原则
- **不简单合并数据**：保留"同一参数在不同偏移下的表现差异"这一最宝贵的反过拟合信息
- **稳健性优先于绝对收益**：宁愿选择5个偏移下都稳定盈利15%的参数，也不选择1个偏移赚50%、4个偏移亏10%的参数
- **双重鲁棒性验证**：同时验证"参数空间鲁棒性"(原L4邻域测试)和"时间空间鲁棒性"(新增偏移测试)

4. **构建参数-偏移矩阵**：为每个参数组合创建一个包含5个偏移所有绩效指标的结构化对象，作为后续所有筛选的基础输入

- 原`top1_pnl_ratio <= 0.45` → 升级为`所有偏移下的top1_pnl_ratio均值 <= 0.45，且最大值 <= 0.55`
- 原`asset_hhi <= 0.65` → 升级为`所有偏移下的asset_hhi均值 <= 0.65，且最大值 <= 0.75`
- 原`profit_loss_ratio >= 0.9`(空头) → 升级为`所有偏移下的profit_loss_ratio均值 >= 0.9，且最小值 >= 0.7`

**核心逻辑**：允许个别偏移下出现稍高的集中度，但不允许平均水平过高，避免参数在不同偏移下过度依赖单一币种或单一交易

### 2.5 L4 邻域平原测试层升级
将原来的"单一偏移下的参数空间鲁棒性"升级为"双重鲁棒性联合测试"：
1. **参数主目标值重构**：对于每个参数组合，不再使用单一偏移下的主目标值，而是使用`5个偏移下主目标值的中位数`作为其真实主目标值
2. **邻域样本量扩大**：将每个参数的5个偏移结果都视为独立的邻域样本，参与邻域健康率和绩效统计计算(样本量直接扩大5倍，统计显著性大幅提升)
3. **邻域健康率升级**：
   - 原`邻域存活率 = 邻域内L1通过参数数 / 邻域总参数数`
   - 升级为`邻域存活率 = 邻域内所有参数-偏移组合中L1通过的数量 / 邻域总参数-偏移组合数`
4. **新增偏移鲁棒性惩罚项**：如果某个参数的5个偏移下主目标值的变异系数(CV) > 0.3，则其邻域健康率乘以0.7的惩罚系数

### 2.6 L5 时间跨度验证层升级
将原来的"单一偏移下的年度稳定性"升级为"多偏移下的跨年度一致性"：
1. **年度盈利比例升级**：
   - 原`盈利年份比例 >= 阈值` → 升级为`平均盈利年份比例 >= 阈值，且最差偏移下的盈利年份比例 >= 阈值的80%`
2. **新增年度偏移一致性检查**：
   - 对于每个年份，要求该参数在至少3个偏移下实现盈利
   - 不允许出现"某一年在所有偏移下都亏损"的情况(除非该年是极端牛市对于空头策略)
3. **新增跨年度偏移相关性**：计算参数在不同年份下的偏移绩效排名相关性，要求相关性 ≥ 0.6(说明参数的偏移表现在不同年份具有一致性)

### 2.7 L6 多偏移一致性验证层(全新新增)
这是专门针对时间偏移过拟合的终极过滤层，只有通过这一层的参数才能进入最终候选池：
| 验证指标 | 计算逻辑 | 建议阈值 | 说明 |
|---------|----------|----------|------|
| 偏移全胜率 | 5个偏移中总收益为正的数量 / 5 | ≥ 0.8(即至少4个偏移盈利) | 绝对底线，不允许出现多数偏移亏损的情况 |
| 绩效衰减率 | (最好偏移总收益 - 最差偏移总收益) / 最好偏移总收益 | ≤ 0.35 | 最好和最差偏移的收益差不能超过35% |
| 最大回撤一致性 | 最差偏移最大回撤 / 最好偏移最大回撤 | ≤ 1.5 | 最差偏移的回撤不能超过最好偏移的1.5倍 |
| 排名稳定性 | 参数在5个偏移下的总收益排名的标准差 | ≤ 0.15 | 参数在不同偏移下的排名不能波动过大 |
| 方向一致性 | 所有偏移下的总收益符号是否一致 | 必须全部一致 | 不允许出现"有的偏移赚有的偏移亏"的情况 |

**淘汰规则**：只要有一项指标不满足阈值，直接淘汰该参数组合

## 三、最终得分计算体系升级
将原来的"邻域LCB得分"升级为"双重鲁棒性综合得分"，公式如下：

```
最终得分 = 邻域稳健性得分 × 时间偏移稳健性得分
```

其中：
1. **邻域稳健性得分**(原逻辑升级)：
   ```
   邻域LCB = 邻域主目标中位数 - 1.5 × 邻域主目标标准差
   邻域稳健性得分 = 邻域LCB × 邻域存活率
   ```
   (主目标值使用5个偏移的中位数)

2. **时间偏移稳健性得分**(新增)：
   ```
   偏移LCB = 5个偏移主目标均值 - 1.5 × 5个偏移主目标标准差
   偏移稳健性得分 = max(偏移LCB, 0) × 偏移全胜率 × (1 - 绩效衰减率)
   ```

**核心优势**：
- 只有同时在"参数空间"和"时间空间"上都表现稳健的参数才能获得高分
- 任何一个维度的脆弱性都会导致最终得分大幅下降

### 核心哲学：从“单点寻优”到“分布评估”
同一组参数在5个Offset下，不应看作5个独立的结果，而应看作**一个随机变量的5个样本点**。一个真正不拟合的好参数，必须满足两个条件：
1. **绝对性能好**（5个Offset的平均收益/风险指标达标）。
2. **相位敏感度低**（5个Offset的方差/波动极小，没有悬崖式的坍塌）。

### 评估框架升级方案（推荐采用“融合降维 + 漏斗升级”模式）

为了最大程度复用你现有的 5 层漏斗，我建议在进入 L1 之前，增加一个 **L0 层（相位融合层 Phase Aggregation）**。具体设计如下：

#### 🚀 L0 层：时间相位融合（Phase Aggregation）
将同一个参数组合的5行数据，合并为1行**“超参数体（Meta-Parameter）”**。对不同类型的字段采取不同的融合逻辑：

1. **收益/风险类指标**（如 `annual_return`, `sortino_ratio`, `calmar_ratio`, `bull/bear_regime_return`）
   - **取均值（Mean）**：代表该参数的期望表现。
   - **取标准差（Std）或变异系数（CV）**：代表“相位敏感度”。
   - *合成新指标*：`Phase_LCB = Mean - 1.5 * Std`。以这个下置信界作为后续评价的核心。
2. **底线生存类指标**（如 `max_drawdown`, `mae_pct_worst`, `max_time_under_water`）
   - **取最劣值（Worst/Min）**：遵循“木桶原理”，即使有4个Offset回撤只有20%，但某个Offset碰巧踩中极端反转导致回撤40%，这组参数的底线回撤就是40%。我们必须用最差的情况来做健康检查。
3. **统计计数类指标**（如 `total_closed_trades`, `active_assets`）
   - **取均值（Mean）** 或 **取中位数（Median）**。
4. **集中度指标**（如 `asset_hhi`, `top1_pnl_ratio`）
   - **取均值（Mean）**。
5. **胜率/年份盈利比**（如 `win_rate`, `profitable_years_ratio`）
   - **取最劣值（Worst）** 结合 **均值（Mean）**。

#### 🛡️ L1 ~ L5 漏斗逻辑的对应演进

完成 L0 层的融合后，我们依然沿用你的 5 层漏斗，但判定标准会发生质变：

*   **L1 硬生存底线（Health）**：
    *   此时的判断基于**L0融合后的最劣值**。例如 `max_drawdown_worst` 不能跌破 -0.35。这大大提高了生存门槛，过滤掉了“仅在特定4小时切分下才存活”的脆弱参数。
*   **L2 Pareto 前沿（Pareto）**：
    *   帕累托寻优的目标改为 `Phase_LCB`（考虑到不同Offset波动的下置信界）。
*   **L3 软约束（Constraints）**：
    *   增加一个针对相位的硬约束：**“最大相位变异系数限制 (Max Phase CV)”**。例如，同一个参数5个Offset的年化收益不能相差超过 30%。如果一个参数 0h 赚 50%，1h 亏 10%，直接在这一层被淘汰。
*   **L4 邻域平原（Neighborhood）**：
    *   *现在的 L4 是：参数A是否处于周围参数（A+1, A-1）的平原中。*
    *   *融合后的 L4 是：参数A的“相位均值和相位下置信界”是否处于周围参数的平原中。*
    *   这实现了终极的平原测试：**“参数微调不敏感 + 进场时间微调不敏感”**。
*   **L5 时间跨度验证（Time）**：
    *   要求“5个Offset的盈利年份比例的最劣值”达标。

#### 🏆 终极评分公式 (Final Score) 的重构
你目前的得分是：`FINAL_SCORE = (邻居均值 - 1.5 * 邻居标准差) * 邻居存活率`
融入 Offset 后，我们可以构建一个更加坚不可摧的评分公式，惩罚两种方差（参数方差 + 相位方差）：

> **最终下置信界** = 均值（5个Offset × N个邻居组成的矩阵） - K * 综合标准差
> **最终得分** = 最终下置信界 × (参数空间存活率 × 相位空间存活率)

### 核心评估哲学：从“单点评估”升级为“分布评估”
对于任何一组给定的策略参数（如 `MOM=36, VOL=90`），它不再只有 1 组绩效，而是有 **5 组绩效**（构成一个微型分布）。好参数的标准变更为：**不仅要在参数的“空间邻居”中表现平稳，还要在“时间偏移”的切片中表现一致。**

### 方案架构设计：时空双重漏斗系统

#### 第一步：数据聚合层（Data Fusion & Transformation）
在进入你现有的 L1 ~ L5 漏斗之前，首先要把同一个参数的 5 份 Offset 数据进行**降维聚合**。针对不同的指标，采用不同的聚合态度：

1. **底线风险指标（悲观原则 - 取最差）：**
   * 最大回撤 (`max_drawdown`)：取 5 个 Offset 中的**最小值（最深回撤）**。
   * 最差 MAE (`mae_pct_worst`)：取 5 个 Offset 中的**最小值**。
   * 单笔最大亏损占比 (`top1_pnl_ratio`)：取 5 个 Offset 中的**最大值**。
   * *逻辑：只要在任何一个时间切片里会爆仓或重伤，这个参数就不能用。*
2. **收益与期望指标（中庸原则 - 取均值/中位数）：**
   * 年化收益、总收益、主目标（Sortino/Calmar）：取 5 个 Offset 的**均值**或**中位数**。
   * 胜率、盈亏比、期望下界：取均值。
3. **新增的衍生指标（稳定性原则 - 算方差）：**
   * **时间偏移变异系数（Offset_CV）**：计算这 5 个主目标（如 Sortino）的标准差除以均值。
   * **时间偏移盈利胜率（Offset_Win_Rate）**：这 5 个 Offset 中，有几个是最终盈利的？（满分5/5）

#### 第二步：升级现有的 L1 ~ L5 漏斗

有了聚合后的数据，你现有的漏斗逻辑可以无缝升级，并变得极其强大：

* **【新增】L0_TEMPORAL (时间鲁棒性过滤)：**
  * **规则 1：** `Offset_Win_Rate >= 4/5`（即在不同的时间点执行，至少 80% 的情况必须是赚钱的，杜绝只在特定时间赚钱的参数）。
  * **规则 2：** `Offset_CV <= 某个阈值`（如果不同开盘时间导致收益上下剧烈波动，说明极度时间过拟合，直接淘汰）。
* **L1_HEALTH (生存底线)：**
  * 现有的 `max_drawdown_threshold` 直接作用于聚合后的**“最深回撤”**。这意味着参数不仅要活下来，而且在最倒霉的进场时机下也要活下来。
* **L2_PARETO (帕累托前沿)：**
  * 基于聚合后的**“平均 Sortino/Calmar”**和**“最差回撤”**去打擂台。
* **L3_CONSTRAINTS (软约束)：**
  * 继续按照原有逻辑执行。
* **L4_NEIGHBORHOOD (空间参数平原)：**
  * **这一步迎来质变！** 以前 L4 是算周围参数的均值。现在是：计算周围参数的**“时间平均表现”**的均值。这样过滤出来的平原，不仅参数稍微偏移不敏感，连执行时间稍微偏移也不敏感。
* **L5_TIME (跨年稳定性)：**
  * 基于 5 个 Offset 的平均年度胜率来判定。

#### 第三步：终极评分公式重构（Double LCB 下置信界）

你现在的 `FINAL_SCORE = 均值 - 1.5 * 标准差` 已经非常棒了（基于 L4 空间邻域）。
现在我们拥有了“空间（参数）方差”和“时间（Offset）方差”，评分公式可以升级为二维的惩罚模型：

* **最终评分 (Spatio-Temporal LCB) =**
  $$ 综合均值 - \alpha \times 空间标准差(邻居间) - \beta \times 时间标准差(Offsets间) $$
* **业务含义：**
  * 综合均值：该参数在不同时间点、不同微调参数下的平均预期。
  * 空间惩罚：参数稍微改变，绩效会崩吗？（防参数过拟合）
  * 时间惩罚：晚半小时入场，绩效会崩吗？（防微观行情过拟合）

---

### 这种方案的巨大优势

1. **分离运气与实力：** 截面动量（CSM）策略本质上是低频换仓。如果你设定每 4 小时换仓，00:00 换仓刚好躲过了一根 03:00 的大阴线，这是运气。用 5 个 Offset 融合评估，把运气的成分强制剥离了。
2. **真实滑点与执行延迟的模拟：** 加密市场波动极大。实盘中，API 延迟、网络问题、计算耗时可能导致你并不是在绝对的 00:00:00 成交。Offset 数据天生包含了这种“执行时差”的宽容度测试。
3. **空头生存力的大考：** 你的 SHORT 引擎在熊市做空。不同时间偏移下，熊市反弹（Short Squeeze）的插针可能会打掉某一两个 Offset 的止损。利用“取最差回撤”原则，能选出真正抗插针的做空参数。

## 一句话结论

**不要把 5 个 offset 简单拼成一张大表直接做筛选。**
正确做法是：**先按“参数组合”分组，再在每个 offset 上独立跑一遍漏斗，最后对同一参数的 5 份结果做跨 offset 聚合评价。**

## 你现在的漏斗，缺了什么

你现有的漏斗已经很强了，核心上是：

1. **L1**：硬生存底线
2. **L2**：Pareto 前沿
3. **L3**：软约束偏好
4. **L4**：邻域平原测试
5. **L5**：时间稳定性

但它缺少一层最关键的东西：

> **“相位稳定性 / offset 鲁棒性”**

因为你的 4h K 线，offset = 0h / 30min / 1h / 2h / 3h，本质上是在测试：

* 信号是否依赖某个特定 bar 截断点；
* 指标窗口是否对采样边界敏感；
* 资金费率、开平仓时刻、趋势切换点是否被某个时点“碰巧对齐”；
* 参数是否只在某个相位上特别好。

这类过拟合很隐蔽，单一 offset 看不出来。

## 推荐的新框架：两层稳健性

我建议你把最终评价拆成两条主线：

### A. 单 offset 内稳健性

就是你现有的 L1~L5。
它回答的是：**“在这个 offset 的世界里，这个参数靠谱么？”**

### B. 跨 offset 稳健性

这是新增的核心层。
它回答的是：**“换一个 bar 起点，它还靠谱么？”**

最终只保留两条线都过关的参数。

## 推荐的完整评价架构

### 第 0 层：数据标准化

先把 5 份结果统一成同一种结构：

* `param_name` 作为主键
* `offset` 作为维度
* 每个 offset 对应一组完整指标

最终形成一个三维评价对象：

> `参数 × offset × 指标`

这是后面所有判断的基础。

---

### 第 1 层：每个 offset 单独跑现有漏斗

对每个 offset，各自执行你现在的：

* L1 健康
* L2 Pareto
* L3 约束
* L4 邻域平原
* L5 时间稳定

这一层不改你的逻辑，只是重复跑 5 次。

你会得到每个参数在 5 个 offset 下的：

* 是否通过
* 各层原因
* 各项指标
* FINAL_SCORE

---

### 第 2 层：跨 offset 的“准入门槛”

这里开始真正选“抗过拟合参数”。

对同一个参数，把 5 个 offset 的结果聚合后，至少看这些东西：

#### 1）通过率

这个参数在 5 个 offset 中，通过最终漏斗的次数。

建议思路：

* 长线多头：至少 **4/5** 通过更合理
* 空头：因为本来更难稳定，可先设 **3/5 或 4/5**，但不能太松

#### 2）主目标中位数

不要只看平均值，优先看 **median**。
因为 5 个 offset 很少，median 比 mean 更抗异常点。

* LONG：看 `sortino_ratio` 或 `FINAL_SCORE` 的中位数
* SHORT：看 `calmar_ratio` 或 `FINAL_SCORE` 的中位数

#### 3）最差 offset 表现

一定要看 **worst-case**，否则容易选到“4 个 offset 很好，1 个 offset 崩掉”的参数。

建议至少记录：

* 最差 offset 的主目标
* 最差 offset 的最大回撤
* 最差 offset 的年化收益
* 最差 offset 的最终 LCB

#### 4）跨 offset 波动

看标准差、极差、IQR 都可以，优先看：

* `std(primary_obj)`
* `std(FINAL_SCORE)`
* `max(primary_obj) - min(primary_obj)`

波动越大，说明参数越吃相位。

---

### 第 3 层：跨 offset 的一致性指标

这一层非常重要，它比单纯收益更能识别“假强参数”。

#### 1）方向一致性

同一个参数在 5 个 offset 下：

* LONG：牛市段是否大多为正？
* SHORT：熊市段是否大多为正？

不是要求 5/5 全正，但至少要“多数一致”。

#### 2）收益符号一致性

看每个 offset 的：

* 总收益符号
* 年化收益符号
* bull / bear regime 收益符号

如果一个参数在 5 个 offset 里正负乱跳，说明它很可能在“吃 bar 对齐红利”。

#### 3）排名稳定性

把同一 offset 下的参数按主目标排序，观察同一个参数的 rank 变化。

重点看：

* 是否经常 Top 10 / Top 20
* 是否 rank 波动很大
* 是否只在某一个 offset 突然冲到第一

这个指标非常能识别“相位依赖型参数”。

---

### 第 4 层：跨 offset 的鲁棒评分

我建议你不要只用一个线性平均分，而是用“**中心表现 + 尾部风险 + 波动惩罚**”的组合。

可以理解为：

> 最终分数 = 中位表现
> − 波动惩罚
> − 最差 offset 惩罚

概念上可以写成：

* **中心项**：5 个 offset 的 median(FINAL_SCORE)
* **稳定项**：减去 std(FINAL_SCORE) 或 MAD
* **尾部项**：再扣最差 1 个或最差 2 个 offset 的损失

这样选出来的参数通常不会“某一根偏移下就失效”。

---

### 第 5 层：跨 offset 的时间稳定性

你现在已经有 year 级别拆分了，这个很好，但建议再升级成：

#### “year × offset” 双维稳定性矩阵

也就是看：

* 每个年份，在 5 个 offset 下是否稳定
* 同一年是否因为 bar 起点不同而完全翻面

理想参数应表现为：

* 多数年份在多数 offset 下为正
* 不会出现某个年份只有一个 offset 非常好，其他全崩

这个层面对“结构性过拟合”特别敏感。

---

## 最终建议的选参规则

我建议你把最终筛选改成“**硬门槛 + 鲁棒排序**”两段式。

### 第一段：硬门槛

参数必须满足：

1. 在 5 个 offset 中，至少有足够多的 offset 通过 L1~L5
2. 最差 offset 不能崩得太离谱
3. 关键约束在多数 offset 上成立
4. 主目标在多数 offset 上为正

这一步是“排除明显不稳的参数”。

### 第二段：鲁棒排序

在剩下的参数里，按下面顺序排：

1. **跨 offset 的 median(FINAL_SCORE)**
2. **跨 offset 的最差 FINAL_SCORE**
3. **跨 offset 的 std(FINAL_SCORE)**
4. **rank 稳定性**
5. **集中度风险稳定性**（HHI、top1_pnl_ratio）
6. **成本压力稳定性**
7. **年际稳定性**

这样排出来的，不是“单点最强”，而是“整体最稳”。

它们不是 5 份独立样本，而是：

* 相位敏感性测试

这意味着你可以用它们来：

* 排除“只在某个切点上好看”的参数

**先在每个 offset 内跑现有漏斗，再把同一参数在 5 个 offset 的结果做聚合，最后只保留“多数 offset 通过 + 中位数强 + 最差情况不崩 + 排名稳定”的参数。**

这会比你现在单 offset 选参，抗过拟合能力强很多，尤其适合你这种：

* 4 小时级别 K 线
* 动态年度池
* 截面动量
* 带资金费率
* 同时存在 long/short 两套引擎

如果你下一步要，我可以继续把这个方案整理成一版**“跨 offset 评价总框架”**的可执行清单，直接对应到你现有的 L1~L5 漏斗结构里。
