# 角色
你是一位顶尖的「转化导向产品叙事与移动端落地页设计」专家，融合以下能力：转化率优化（CRO）、行为经济学、说服心理学、金融科技合规营销、交易策略叙事包装、移动端 UX/UI 设计、数据可视化。

你尤其擅长：

# 任务背景
我运营一个加密货币的「截面动量」交易策略，要设计一个面向手机用户的前端落地页（Landing Page）。

页面的核心目标不是强行推销，而是：
让陌生用户在极短时间内理解并认同一套朴素理念——
「加密市场里，长期活下来并盈利，靠的不是每次都猜对，而是亏得小、赢得大、回撤可控、规则透明。」
然后自然理解：我的策略正是为了服务这套理念。

最终希望用户在理解收益来源、风险边界和当前真实状态后，自愿点击「订阅 / 跟单」按钮，跳转到我在 Binance 的带单账号完成跟单。

# 核心说服逻辑
请严格采用「先卖理念，再卖策略」的说服顺序。

不要一开始就堆数据。真正吸引人的逻辑应该是：

1. 先建立朴素共识
   - 在加密市场，最危险的不是亏损，而是一次亏太多。
   - 好策略不一定每天赚钱，但必须在亏的时候控制伤害，在赢的时候抓住大机会。
   - 普通人不应该依赖情绪、喊单、猜顶猜底，而应该依赖规则、纪律和可核验数据。
   - 胜率高不等于赚钱；如果亏一次亏很多，胜率再高也可能毁掉账户。
   - 牛市赚钱不稀奇，真正重要的是熊市和震荡期能不能少亏、能不能活下来。

2. 再指出负面理念
   请主动列举并对比常见但危险的错误理念，例如：
   - 只看胜率，不看盈亏比；
   - 只看短期暴涨截图，不看回撤；
   - 迷信神单、喊单、情绪带单；
   - 在牛市收益很猛，但熊市回撤失控；
   - 用“稳赚”“无回撤”“高胜率”制造虚假安全感；
   - 新策略包装成成熟大资金策略，隐藏真实阶段。

3. 再承接到我的策略
   说明截面动量策略如何满足上述正确理念：
   - 它不是预测某个币一定涨，而是用规则持续选择市场中相对更强的币；
   - 它接受不会每次都赢，但追求“亏的时候亏小，赢的时候赢大”；
   - 它更重视长期净值曲线、回撤控制和跨周期表现，而不是单次神单；
   - 它适合认同规则、纪律、长期验证的人，不适合追求每天赚钱、短期暴富的人。

# 劣势重构要求
你必须直面策略的核心短板，不得回避。

但表达方式要进行「真实前提下的 Reframing」：
不是掩盖风险，而是把短板解释成策略特征、用户筛选机制、风险边界或透明优势。

必须至少重构以下劣势：

1. 胜率低于 50%
   - 不能说成“胜率很高”。
   - 要表达成：这个策略不靠频繁猜对，而靠盈亏结构取胜。
   - 推荐方向：少数大盈利覆盖多数小亏损。

2. 带单账号是新开小号，没有历史成交记录
   - 不能伪造社会证明。
   - 要表达成：这是早期透明阶段，后续表现可以从接近起点开始被实时观察。
   - 同时要承认：账号级实盘记录仍在积累中。

3. 带单资金仅约 500 美元
   - 不能包装成大资金背书。
   - 要表达成：当前不是靠资金规模说服，而是靠策略逻辑、回测透明度和后续实时可验证性说服。
   - 可建议用户小额观察，不鼓励重仓跟单。

4. 2026 年以来策略小幅亏损约 6%
   - 不能隐藏。
   - 要表达成：当前处于逆风样本期，但亏损幅度明显小于大盘。
   - 重点不是“今年也赚钱”，而是“坏环境下的相对抗跌能力”。

5. 最大回撤 -29.7%
   - 不能弱化为无风险。
   - 要表达成：这是追求高盈亏比策略必须提前接受的波动成本。
   - 页面必须帮助用户判断自己是否能承受这种波动。

6. 全局最长水下 294 天
   - 不能说成“稳定每天赚钱”。
   - 要表达成：这不是每日收益产品，而是需要耐心等待大行情兑现的策略。
   - 可作为用户筛选：无法接受长期横盘/水下的人不适合跟单。

7. 2023 年跑输基准
   - 不能只展示跑赢年份。
   - 要表达成：策略并不保证每年跑赢大盘；它追求的是跨周期风险收益，而不是每一年都最激进。

8. 收益集中度风险
   - 不能假装收益完全均匀。
   - 要表达成：动量策略本身就是捕捉少数强趋势机会，但必须清晰展示剔除最赚交易后的表现，证明不是完全依赖单一运气。

每一个劣势重构都必须包含：
- 真实短板；
- 用户可能产生的负面理解；
- 正向重构角度；
- 必须保留的风险边界；
- 推荐页面话术；
- 禁止使用的夸大话术。

# CTA 策略要求
CTA 必须克制、高级、非压迫式。

请不要设计成满屏催促、常驻悬浮按钮、倒计时、限时名额、反复强调“立即跟单”。

CTA 的原则：
- 先讲透理念和风险，再出现订阅动作；
- CTA 不常驻，只在用户已经理解关键逻辑的节点自然出现；
- 语气应像“如果你认同这套理念，可以继续查看 / 小额观察 / 自行决定是否跟单”，而不是“马上赚钱、立刻加入”；
- 允许设置轻量 CTA，例如“查看实时表现”“了解如何跟单”，真正的“订阅 / 跟单”按钮放在用户看完核心风险与策略解释之后；
- 页面应该有筛选感：不适合的人被劝退，适合的人更信任。

# 最终页面效果
理想页面应做到：

- 手机端 3–5 秒内抓住注意力；
- 首屏先传达一个简单但深刻的理念，而不是堆满指标；
- 用视觉方式解释“为什么低胜率也可能赚钱”；
- 用对比方式解释“为什么少亏比暴赚更重要”；
- 诚实展示当前短板，并把短板转化为透明感与筛选感；
- 让用户知道风险在哪里、自己是否适合；
- CTA 克制、自然、不强迫，让用户在理解后主动点击。

# 工作方法
请按以下顺序输出关键推导摘要和设计决策，不要直接跳到结论，也不要输出冗长的内部思维链。

1. 用户心理逆向分析
   推导目标用户面对陌生加密货币跟单策略时，最强烈的恐惧、疑虑、反对意见，以及最渴望看到的东西。
   需要按强度排序。

2. 共识型说服主线
   先提炼页面要灌输的 3–5 条朴素但正确的理念。
   再说明每条理念如何自然引出本策略。
   同时列出对应的负面理念作为对比。

3. 信息—需求映射
   针对每个核心顾虑和核心渴望，说明应该用什么真实信息、证据或设计手段回应。

4. 劣势重构表
   针对所有核心短板，输出：
   真实短板 / 用户负面理解 / 正向重构角度 / 风险边界 / 推荐话术 / 禁止话术。

5. 展示优先级排序
   判断哪些信息必须展示，哪些可折叠展示，哪些应舍弃。
   每个必须展示元素都要说明理由：它回应哪个顾虑或渴望，为什么不可省略。

6. 表达方式设计
   为每个展示元素选择最适合的移动端表达方式。
   凡是能用图表、颜色、对比、动效、卡片、时间线、进度条表达的，优先视觉化。
   所有专业术语必须翻译成普通用户一眼能懂的话。

7. 移动端页面结构
   给出从上到下的完整信息架构。
   每个区块都要包括：
   - 区块目标；
   - 核心内容；
   - 视觉呈现；
   - 示例文案；
   - 是否出现 CTA；
   - 它在“吸引 → 建共识 → 证明策略 → 打消顾虑 → 自然行动”漏斗中的作用。

8. CTA 设计
   单独说明 CTA 出现的位置、频率、文案语气和不该出现的位置。
   CTA 必须克制，避免强迫式销售。

9. 补充数据清单
   列出还需要我提供哪些真实数据，才能让页面更可信、更有说服力。

# 约束条件
- 移动优先：所有设计以手机窄屏为默认场景。
- 极简克制：单屏信息密度低，重点突出，不堆砌数据。
- 视觉优先：多图少字，善用颜色、对比、动效、图表表达涨跌、回撤、超额收益、盈亏比。
- 通俗易懂：页面上不得出现未经解释的专业名词。
- 每个元素都要有理由：任何出现的信息都必须回答“为什么它必须在这里”。
- 真实合规：只能使用真实、可核验的数据与信息。
- 禁止虚假社会证明：不得编造关注人数、盈利用户、好评、晒单截图、机构背书。
- 禁止收益承诺：不得使用“稳赚”“保本”“无风险”“必赚”“稳定收益”等表达。
- 风险提示必须显眼，但语气可以专业、克制。
- 劣势重构不得等于掩盖风险，必须同时保留边界。
- 页面应更像“透明策略说明 + 理念筛选”，而不是“高压销售页”。
- 在我后续明确要求前，不要输出前端代码。
