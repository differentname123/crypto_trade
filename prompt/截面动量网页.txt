# 任务背景
我运营一个加密货币的「截面动量」交易策略，要设计一个面向手机用户的前端落地页（Landing Page）。

页面的核心目标不是强行推销，而是：
让陌生用户在极短时间内理解并认同一套朴素理念——
「加密市场里，长期活下来并盈利，靠的不是每次都猜对，而是亏得小、赢得大、回撤可控、规则透明。」
然后自然理解：我的策略正是为了服务这套理念。

最终希望用户在理解收益来源、风险边界和当前真实状态后，自愿点击「订阅 / 跟单」按钮，跳转到我在 Binance 的带单账号完成跟单。

# 核心说服逻辑
请严格采用「先卖理念，再卖策略」的说服顺序。

不要一开始就堆数据。真正吸引人的逻辑应该是：

1. 先建立朴素共识
   - 在加密市场，最危险的不是亏损，而是一次亏太多。
   - 好策略不一定每天赚钱，但必须在亏的时候控制伤害，在赢的时候抓住大机会。
   - 普通人不应该依赖情绪、喊单、猜顶猜底，而应该依赖规则、纪律和可核验数据。
   - 胜率高不等于赚钱；如果亏一次亏很多，胜率再高也可能毁掉账户。
   - 牛市赚钱不稀奇，真正重要的是熊市和震荡期能不能少亏、能不能活下来。

2. 再指出负面理念
   请主动列举并对比常见但危险的错误理念，例如：
   - 只看胜率，不看盈亏比；
   - 只看短期暴涨截图，不看回撤；
   - 迷信神单、喊单、情绪带单；
   - 在牛市收益很猛，但熊市回撤失控；
   - 用"稳赚""无回撤""高胜率"制造虚假安全感；
   - 新策略包装成成熟大资金策略，隐藏真实阶段。

3. 再承接到我的策略
   说明截面动量策略如何满足上述正确理念：
   - 它不是预测某个币一定涨，而是用规则持续选择市场中相对更强的币；
   - 它接受不会每次都赢，但追求"亏的时候亏小，赢的时候赢大"；
   - 它更重视长期净值曲线、回撤控制和跨周期表现，而不是单次神单；
   - 它适合认同规则、纪律、长期验证的人，不适合追求每天赚钱、短期暴富的人。

> 注：以上规定的是说服的**内容与顺序**。这些朴素理念最终如何被"凝练成主题、表达成金句"，必须严格遵循下方「核心主题与理念凝练要求」的气质标准——朴素的是道理，高级的是表达。

# 核心主题与理念凝练要求（最高优先级）

在展开任何说服逻辑之前，你必须先完成两件事：**凝练一个核心主题**，并**提炼一组理念金句**。它们是整个页面的灵魂——用户最先看到的是主题，再通过理念被层层说服，最后才理解策略。务必先想透这两样，再设计其余内容。

## 一、核心主题（页面"题眼"）
- 你必须凝练出一句统领全局的核心主题，所有理念、所有策略解释都围绕它展开。
- 主题必须是一句**金句**：有冲击力、抓人眼球、能在 3 秒内击中认知，让用户产生"原来如此"的顿悟感。
- 主题应揭示一个**反直觉、但一旦想通就无法反驳**的市场真相，而不是一句正确的废话。
- 用户先看到主题、再读理念——主题为后续所有内容提供理解框架，因此它必须精准、有分量。
- 风格参照方向（须再优化，禁止照抄）：
  · "穿越周期的，从来不是预测，而是结构。"
  · "决定终局的，不是胜率，而是赔率。"
  · "市场无法预测，但盈亏可以被设计。"

## 二、理念金句（3–5 条，主题的展开）
- 在主题之下提炼 3–5 条朴素但正确的理念，作为主题的支撑与延伸。
- 每条理念都须同时满足：**朴素易懂**（普通人一眼能懂）＋**暗含深度**（说出多数人没想透的道理）＋**有煽动性**（让人忍不住点头认同）。
- 理念之间应层层递进，建议沿"市场的本质 → 风险的本质 → 盈利的本质 → 周期的本质 → 透明的本质"展开。
- 每条理念都要能**自然引出本策略的某个特征**，不生硬、不自夸。

## 三、用词与气质标准（关键）
主题与理念的表达，必须达到以下气质：
- **克制**：不喊口号、不打鸡血、不堆叹号，越平静越有分量。
- **高级**：像顶级量化机构或资深投资人的语气——冷静、精准、笃定。
- **学术美感**：可调用"非对称、结构、期望值、复利、周期"等高阶概念，但必须用普通人能懂的话说出来；高级感来自**思想的深度**，而非术语的堆砌。
- **降维说服力**：让用户感到自己被带到更高的认知维度，看清了多数人看不清的东西。

## 四、严禁的低级表达（重点）
以下话术会让策略显得廉价、业余、缺乏技术含量，一律禁止：
- **禁止"意志力叙事"**：不得把策略优势归因于"认错、纪律、忍耐、扛住"等意志力层面——这会让策略显得被动硬扛、毫无结构性优势。优势必须归因于**结构、系统、概率分布**等可复制的层面。
- **禁止戏剧化 / 爽文 / 街头气词汇**：如"冷酷的概率猎手""收割""暴富""碾压""信仰""杀疯了"等。
- **禁止居高临下的说教**：不要指责用户"别幻想""你太天真"——要让用户自己顿悟，而非被教训。

正反对照（请严格遵循右侧气质）：
- ❌ "靠'认错'活下来，靠'纪律'赚大钱"
  ✅ "我们不靠每次猜对取胜，而靠盈亏的量级结构取胜——亏损被结构性地限制，盈利被结构性地放大。"
  （把"被动硬扛"改写为"主动设计的结构优势"）
- ❌ "放弃'全胜'幻觉，做冷酷的'概率猎手'"
  ✅ "你不需要每次都对；只要对的时候，足够重要。"
  （去掉中二营销腔，用克制语言说出非对称收益）

# 劣势重构要求
你必须直面策略的核心短板，不得回避。

但表达方式要进行「真实前提下的 Reframing」：
不是掩盖风险，而是把短板解释成策略特征、用户筛选机制、风险边界或透明优势。所有重构话术同样须符合「核心主题与理念凝练要求」的气质标准（克制、高级、不靠意志力叙事）。

必须至少重构以下劣势：

1. 胜率低于 50%
   - 不能说成"胜率很高"。
   - 要表达成：这个策略不靠频繁猜对，而靠盈亏结构取胜。
   - 推荐方向：少数大盈利覆盖多数小亏损。

2. 带单账号是新开小号，没有历史成交记录
   - 不能伪造社会证明。
   - 要表达成：这是早期透明阶段，后续表现可以从接近起点开始被实时观察。
   - 同时要承认：账号级实盘记录仍在积累中。

3. 带单资金仅约 500 美元
   - 不能包装成大资金背书。
   - 要表达成：当前不是靠资金规模说服，而是靠策略逻辑、回测透明度和后续实时可验证性说服。
   - 可建议用户小额观察，不鼓励重仓跟单。

4. 2026 年以来策略小幅亏损约 6%
   - 不能隐藏。
   - 要表达成：当前处于逆风样本期，但亏损幅度明显小于大盘。
   - 重点不是"今年也赚钱"，而是"坏环境下的相对抗跌能力"。

5. 最大回撤 -20.5%
   - 不能弱化为无风险。
   - 要表达成：这是追求高盈亏比策略必须提前接受的波动成本。
   - 页面必须帮助用户判断自己是否能承受这种波动。

6. 全局最长水下 254 天
   - 不能说成"稳定每天赚钱"。
   - 要表达成：这不是每日收益产品，而是需要耐心等待大行情兑现的策略。
   - 可作为用户筛选：无法接受长期横盘 / 水下的人不适合跟单。

7. 2023 年跑输基准
   - 不能只展示跑赢年份。
   - 要表达成：策略并不保证每年跑赢大盘；它追求的是跨周期风险收益，而不是每一年都最激进。

8. 收益集中度风险
   - 不能假装收益完全均匀。
   - 要表达成：动量策略本身就是捕捉少数强趋势机会，但必须清晰展示剔除最赚交易后的表现，证明不是完全依赖单一运气。

每一个劣势重构都必须包含：
- 真实短板；
- 用户可能产生的负面理解；
- 正向重构角度；
- 必须保留的风险边界；
- 推荐页面话术；
- 禁止使用的夸大话术。

# CTA 策略要求
CTA 必须克制、高级、非压迫式。

请不要设计成满屏催促、常驻悬浮按钮、倒计时、限时名额、反复强调"立即跟单"。

CTA 的原则：
- 先讲透理念和风险，再出现订阅动作；
- CTA 不常驻，只在用户已经理解关键逻辑的节点自然出现；
- 语气应像"如果你认同这套理念，可以继续查看 / 小额观察 / 自行决定是否跟单"，而不是"马上赚钱、立刻加入"；
- 允许设置轻量 CTA，例如"查看实时表现""了解如何跟单"，真正的"订阅 / 跟单"按钮放在用户看完核心风险与策略解释之后；
- 页面应该有筛选感：不适合的人被劝退，适合的人更信任。

# 工作方法

# 约束条件
- 移动优先：所有设计以手机窄屏为默认场景。
- 极简克制：单屏信息密度低，重点突出，不堆砌数据。
- 视觉优先：多图少字，善用颜色、对比、动效、图表表达涨跌、回撤、超额收益、盈亏比。
- 通俗易懂：页面上不得出现未经解释的专业名词。
- **主题先行：页面必须有一句统领全局的核心主题金句，且为用户最先看到的内容。**
- **话术高级克制：所有主题、理念与文案必须达到机构级的克制与学术质感，严禁戏剧化、爽文式、街头气、说教式表达（详见「核心主题与理念凝练要求」）。**
- **优势归因于结构而非意志力：严禁把策略优势包装成"靠认错、靠纪律、靠忍耐、靠扛"等意志力叙事；必须落到结构性、系统性、概率分布层面的可复制优势。**
- 每个元素都要有理由：任何出现的信息都必须回答"为什么它必须在这里"。
- 真实合规：只能使用真实、可核验的数据与信息。
- 禁止虚假社会证明：不得编造关注人数、盈利用户、好评、晒单截图、机构背书。
- 禁止收益承诺：不得使用"稳赚""保本""无风险""必赚""稳定收益"等表达。
- 风险提示必须显眼，但语气可以专业、克制。
- 劣势重构不得等于掩盖风险，必须同时保留边界。
- 页面应更像"透明策略说明 + 理念筛选"，而不是"高压销售页"。
- 在我后续明确要求前，不要输出前端代码。
