# 最终产出要达成的效果（以终为始）
理想的页面应当做到：在用户 3–5 秒的极短注意力窗口内抓住眼球；用最少的文字、最多的视觉表达，说清楚"这个策略凭什么值得信任、能带来什么、风险在哪、怎么开始"；逐一化解目标用户内心最深的顾虑；最终让用户自然且有意愿地点击订阅。整个体验必须在手机窄屏上极其顺畅。

# 工作方法（请严格按以下顺序推理，并完整展示你的推理过程，不要直接跳到结论）
1. 用户心理逆向分析：先独立、尽可能穷尽地推导出——目标用户（以散户、移动端、注意力极短的人群为主）在面对"一个陌生的加密货币跟单策略"时，内心最强烈的恐惧、疑虑与反对意见是什么；以及他们最渴望看到、最能被打动的是什么。按强烈程度排序。不要被我上面列出的数据限制你的思考范围，你要从"如何赢得一个陌生人的信任与行动"这个根本问题出发自由推导。
2. 信息—需求映射：针对上一步得出的每一个核心顾虑和每一个核心渴望，推导出"用什么真实的信息、证据或设计手段"能最有效地回应它。
3. 展示优先级排序：考虑到注意力极短、页面必须极简，判定哪些信息是"必须展示"、哪些"次要可选"、哪些"应当舍弃"。为每一个"必须展示"的元素给出明确理由（它回应了哪个顾虑或渴望、为什么不可省略）。严禁为了填充版面而堆砌信息。
4. 表达方式设计：为每个要展示的信息选择最易懂、最直观的表达方式。凡是能用图表、图形、颜色、动效、对比等视觉方式表达的，一律优先视觉化，尽量减少枯燥的大段文字。所有专业术语必须翻译成普通用户一眼能懂的说法。
5. 页面结构设计：基于以上结论，给出移动端从上到下的完整信息架构（逐区块），明确每个区块的内容、视觉呈现方式、以及它在"吸引→建立信任→打消顾虑→促成点击"这条转化漏斗中承担的作用。
6. 补充数据清单：列出还需要我提供哪些真实数据，才能让页面更有说服力。

# 约束条件
- 移动优先：所有设计以手机窄屏为默认场景，而非桌面端。
- 极简克制：尊重极短注意力，单屏信息密度低、重点突出，行动召唤（CTA）始终清晰可达。
- 视觉优先：多图少字，善用颜色、对比、动效来传达涨跌、风险、超额收益等概念。
- 通俗易懂：页面上不得出现任何未经解释的专业名词。
- 每个元素都要有理由：任何出现在页面上的信息，都必须能回答"为什么它必须在这里"。
- 真实与合规（重要约束）：页面只能使用真实、可核验的数据与信息。禁止编造关注人数、盈利用户、好评/晒单截图等虚假社会证明；禁止任何收益承诺或"稳赚不赔"的暗示；必须包含恰当、显眼的风险提示。原因：金融类推广一旦被发现造假，会面临平台封号与法律风险，而真实、可核验的信任信号转化更稳、更经得起用户深究。请在这一前提下，主动寻找真实而有力的信任建立方式（例如但不限于：策略逻辑的透明化、真实回测的可视化、真实风控指标、对"早期阶段/新账号"诚实而正向的定位、可被实时核验的净值表现等）。
